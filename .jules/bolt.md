## 2024-04-18 - [Backend Data Service] Concurrent Pipeline Stalls
**Learning:** Sequential chunking using `Promise.all(chunk)` for bounded concurrency creates artificial pipeline stalls because the entire batch waits for the slowest task to finish before starting the next batch.
**Action:** Implemented a sliding-window promise pool using `Promise.race` and an execution array to keep exactly `CONCURRENCY_LIMIT` tasks in flight at all times. Also, always attach empty `.catch()` handlers to promises immediately upon creation in manual promise pools to prevent `UnhandledPromiseRejection` crashes from background failures before `Promise.all` can observe them.
## 2024-04-24 - [Backend Orchestrator Service] JSON.parse in a loop
**Learning:** Calling `JSON.parse` inside a loop or frequently invoked function like an authorization check can be a significant bottleneck due to O(N) parsing overhead. When using a `WeakMap` cache to prevent this, the returned cached array or object must be shallow-copied (e.g. `[...cache.parsed]`) to avoid accidental shared reference mutation bugs where callers unintentionally modify the cached values.
**Action:** Use a `WeakMap` keyed by the object reference for caching repeated `JSON.parse` operations on shared configuration objects, but ensure that cache entries also account for potential mutations to the underlying string (e.g., store the original string and compare it to bust the cache). Always return a clone of the cached complex data structure to protect cache integrity.
